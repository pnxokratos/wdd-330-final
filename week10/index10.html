Week 10

  • This week's reading is about using Fetch and Client-Side Form Validation.
  • Fetch reading url = https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch
  • Client-Side Form Validation reading url =https://developer.mozilla.org/en-US/docs/Learn/Forms/Form_validation
  • Simple Fetch statement:
    fetch('http://example.com/movies.json')
    .then(response => response.json())
    .then(data => console.log(data));
  • The simplest use of fetch() takes one argument and returns a promise containing the response (a Response object).

    • Sending a request with credentials included

  • To cause browsers to send a request with credentials included, even for a cross-origin call, add credentials: 'include' to the init object you pass to the fetch() method.
  • Files can be uploaded using an HTML input type="file" input element, FormData() and fetch().
  • Files can be uploaded using an HTML input type="file" multiple input element, FormData() and fetch().
  • Both of the "input" and the "multiple" above would go inside the input tag.

    • Checking that a fetch was successful

  • An accurate check for a successful fetch() would include checking that the promise resolved, then checking that the Response.ok property has a value of true.

    • fetch('flowers.jpg')
    .then(response => {
    if (!response.ok) {
    throw new Error('Network response was not ok');
    }
    return response.blob();
    })

    myImage.src = URL.createObjectURL(myBlob);

    Headers

  • The Headers interface allows you to create your own headers object via the Headers() constructor.
  • A headers object is a simple multi-map of names to values.
  • You can combine a lot of this cool info, like Response objects "response.ok" and such, with headers and new Blob's

    • Client-side form validation

  • There's built-in form validation and then there's your own custom JS.
  • Required, min and max length, type, and pattern are all attributes you can use to ensure valid form fields.
  • There's valid and invalid checks.
  • You can add CSS to an invalid input.
