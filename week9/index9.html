Chapter 9 (The Window Object) - Novice to Ninja

  • BOM - Browser Object Model. A quick way of assigning the variable global to the global object looks like this: const global = this;
  • Global variables don't need a const or let or var in front: x = 6;
  • window.alert(), window.confirm() and window.prompt() will stop the browser from running it's program, so use with caution.
  • window.navigator.userAgent tells info about the user's browser, but this may not always be accurate because it can be changed manually by the user.
  • window.location.href, window.location.protocol, window.location.host, window.location.port, window.location.pathname, window.location.search, window.location.hash, window.location.origin, window.location.assign('https://www.sitepoint.com/'), window.location.toString() are all referenced in this chapter. I'm listing them here as a reference point if I need them to go back and look them up.
  • The Browser History - window.history.go(1); // goes forward 1 page
  • A new window can be opened using the window.open() method. This takes the URL of the page to be opened as its first parameter, the window title as its second parameter, and a list of attributes as the third parameter.
  • popup.close(), window.moveTo(0,0); // will move the window to the top-left corner of the screen, window.resizeTo(600,400);
  • More Properties: window.screen.height, window.screen.width, window.screen.availWidth, window.screen.availHeight, window.screen.colorDepth;

    • Cookies

  • document.cookie = 'name=Superman';
  • Cookies are session cookies by default. This means they are deleted once a browser session is finished (when the user closes the browser tab or window).
  • You can set the cookie to expire later than when a user closes the browser, say one day.
  • Example: document.cookie = 'name=Batman; max-age=86400' // 86400 secs = 1 day
  • Adding the string ; secure to the end of a cookie will ensure itâ€™s only transmitted over a secure HTTPS network.
  • To delete a cookie, you need to set it to expire at a time in the past: document.cookie = 'name=Batman; expires=Thu, 01 Jan 1970 00:00:01 GMT';

  • The window.setTimeout() method accepts a callback to a function as its first parameter and a number of milliseconds as its second parameter.

    • Chapter 14 (HTML5 APIs)

  • The data attribute is a way of embedding data into a web page without interferring with the browser. The browser can just ignore the data.
  • Naming the data attributes: must start with "data-".
  • They must contain only lowercase letters, numbers, hyphens, dots, colons or underscores. No uppercase.
  • Include an optional string value.
  • Some examples:
    • data-powers = 'flight superSpeed'
    • data-rating = '5'
    • data-dropdown
    • data-user = 'DAZ'
    • data-max-length = '32'
  • The word "dataset" also seems to be a keyword to accessing the data- attribute.

    • HTML5 Web Storage

  • Using APIs is like using cookies except you can store more information, has fewer restrictions, and is generally easier to use.
  • Web Storage API has some crucial differences with cookies:
    • Information stored isnotshared with the server on every request.
    • Information is available in multiple windows of the browser (but only if the domain is the same).
    • Storage capacity limit is much larger than the 4KB limit for cookies ( There is no actual limit in the specification, but most browsers have a limit set at 5GB per domain.).
    • Any data stored does not automatically expire as it does with cookies. This potentially makes cookies a better choice for something like showing a popup once a day.
  • To set something to a value locally: localStorage.setItem('name', 'Walter White');
  • Rather than using getItem() and setItem(), you can do it like this: localStorage.name = 'Heisenberg';
  • You can delete an element like this: delete localStorage.name;
  • To complete wipe out the LS: localStorage.clear();
  • Here's an example: addEventListener('storage', (event) => {
    console.log(`The ${event.key} was updated from ${event.oldValue} to ${event.newValue} and saved in
    ${event.storageArea}`) }, false);
  • localStorage.setItem('superman', JSON.stringify(hero);
  • This will save the hero object as a JSON string using the string 'superman' as the key. To retrieve the superhero as a JavaScript object:
  • superman = JSON.parse(localStorage.getItem('superman'));
  • Getting a user's location is pretty easy through the geolocation api. "navigator.geolocation.getCurrentPosition(youAreHere);"
  • The position object has several other properties too like position.speed, position.altitude, position.heading, position.timestamp

    • Web Workers! (what a great name!)

  • Web workers run in the background so other things can run in the foreground.
  • Worker constructor function could look like this: const worker = new Worker('task.js');

    • (Chrome browser won't let you run workers from local files like this. A workaround is start Chrome using the --allow-file-access-from-files flag, or simply use a different browser for this example.)

  • Use the postMessage() method to send a message and start a worker working. worker.postMessage('Hello');
  • Use the keyword "self" to send a message from the the worker: self.postMessage('Finished');
  • Using a message allows us to create an event listener, or to use an event listener. Example:
    worker.addEventListener('message', (event) => {
    console.log(event.data);
    }, false);
  • Stop a worker by using "terminate()": worker.terminate();
  • When you are inside the worker, you can use: self.close();
