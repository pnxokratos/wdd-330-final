Chapter 8

Transforms

  • CSS3 allows us to tilt, scale, move, skew, and flip elements!
  • The "transform" property helps us do this.
  • Translation Function - allows you to move up, down, left, right relative to it's current position.
  • Unlike "position: relative", the translate function doesn't matter what its parent object is doing.
  • "translate(x, y)" allows you to move to the left and down from the top
  • If you just want to move right, "transform: translateX(45px);"
  • If you just want to move down, "transform: translateY(45px);"
  • Scale(x, y) can scale the element.
  • You can scale the whole thing bigger by double: scale(2)
  • You can make it smaller: scale(.5)
  • You can make it wider: scaleX(2)
  • You can make it taller: scaleY(2)
  • rotate() turns an element. Here's one example turn by degrees: rotate(10deg)
  • Positive numbers turn clockwise; negative numbers turn counterclockwise.
  • skew(x, y) skews an element along the x and y axes.
  • skewX() and skewY() do what all the other translation properties do.
  • Order of Operations Matters!

    • Transitions

  • The transition property allows you to make changes gradually instead of quickly
  • The list of properties that can be animated is found at https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_animated_properties.
  • Transforms vs. Transitions: transform changes the elements position, scale or skew. Transition changes properties like color, border, etc.
  • Declaration: transition-property: transform;
  • "transition-duration: 0.2s;" sets the time it takes for the transition-duration. .2s is a good amount of time that is not too slow.

    Canvas vs. SVG

  • Canvas is faster, but it can't be added to it once you've created it. You'd have to start over. You can save a drawing to a PNG or JPEG file though.
  • SVG is accessible via the DOM, so if you need to be able to access and change specific aspects of your graphics, SVG might be more appropriate.
